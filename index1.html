Protocol — v1.0

Lipid Polymer Nanoparticles — Preparation & Characterisation

LPN · mRNA Microfluidics DLS · AF4 · SAXS Tina Jurkošek

Clean the bench with RNaseZAP before starting. Wear gloves at all times when handling RNA-containing solutions. Use RNase-free consumables throughout.

1 Buffer preparation ▶

PBS (1.5 L, 20 mM, pH 7.4): Dissolve NaCl (12.0 g), KH₂PO₄ (0.3 g), KCl (0.3 g), and Na₂HPO₄ (1.725 g) in 1.5 L Milli-Q water with magnetic stirring. Bake glass beakers at 180 °C for ≥ 6 h after RNaseZAP treatment.

Citrate buffer (50 mL, 100 mM, pH 4.0): Dissolve sodium citrate dihydrate (248.22 mg) and citric acid (318.2 mg) in RNase-free water up to 50 mL. Sterile-filter through a 0.45 µm membrane.

Verify pH of both buffers the following day with a calibrated pH meter.
2 Stock solutions & organic / aqueous phase ▶

Formulation A — C12-200 / DOPE / PLGA / DMPE-PEG (N/P 10, C12-200:DOPE 3:1, 7 mol% PEG, 50 wt% PLGA). Dissolve C12-200 and DMPE-PEG in 2% v/v acetic acid in acetonitrile; DOPE in absolute ethanol; PLGA in pure acetonitrile — prepare fresh on the day.

Formulation B — SM-102 / DSPC / PLGA / DMG-PEG (N/P 11.8). Same lipid roles and solvents as Formulation A. PLGA still prepared fresh.

Add organic-phase components in order: acetic acid/acetonitrile excess → ionisable lipid → PEG-lipid → helper lipid → PLGA last. Warm all stocks at 50 °C for 2 min and vortex before use. Do not delay after adding PLGA.

Aqueous phase: For mRNA-loaded particles, dilute FLuc mRNA stock to the target concentration in citrate buffer (pH 4.0). For empty particles, replace the mRNA volume with an equal volume of citrate buffer.

3 Microfluidic assembly — Nanoassemblr Ignite ▶
Flow rate ratio (aq:org)3:1
Total flow rate12 mL/min
Start waste volume0.25 mL
End waste volume0.10 mL
Total processed volume12 mL
Remove all air bubbles from both syringes before inserting. Load organic phase in 5 mL syringe, aqueous phase in 10 mL syringe.
4 Dialysis ▶

Dialyse against 1.5 L PBS (Slide-A-Lyzer cassettes): 3 h at room temperature → overnight at 4 °C, both with continuous stirring. Removes residual acetonitrile/ethanol and exchanges citrate (pH 4.0) for PBS (pH 7.4).

Draw 0.3 mL of air into the syringe before aspirating the formulation — this prevents over-swelling of the membrane. Inject through the top port; rinse the vial with a small PBS volume and inject into the cassette too.

Never touch the membrane and never let it dry. Handle cassettes by the rigid frame only.
5 Upconcentration ▶

Centrifugal filtration at 3000 × g, 4 °C. Activate membranes first (5 min spin with PBS, discard flow-through). Add formulation in portions with 15 min spins until ~100 µL remains. Collect into 0.5 mL non-stick hydrophobic microtubes.

Always balance the rotor with two vials. Record the exact recovered volume — required for downstream concentration calculations.
6 Dynamic light scattering (DLS) ▶

Measures hydrodynamic radius (Rh) and PDI via the Stokes-Einstein relation. Target concentration: 0.18 mg/mL in ~700 µL PBS. PDI < 0.2 indicates a monodisperse population.

7 RiboGreen assay ▶

Quantifies mRNA to determine encapsulation efficiency (EE%) and recovery. Two conditions: TE buffer (free RNA) and 2% Triton X-100 in TE (total RNA after LPN disruption).

Standard curve: 1:2 serial dilution from 100 ng/mL in 6 wells, run in triplicate for both TE and TX. Typical dilutions ~1:4 000 (TE), ~1:20 000 (TX). Read plate immediately after adding RiboGreen dye, keeping it protected from light.

TX is viscous — use reverse pipetting. Protect RiboGreen dye from light at all times. Vortex both TE and TX buffers before starting the plate.
8 AF4 fractionation ▶

Dilute to 4 mg/mL (minimum 2 mg/mL) in PBS. Standard injection: 220 µL (200 µL + 20 µL excess). For online AF4-SAXS: same preparation; ≥ 1 L PBS carrier buffer sufficient for 3 samples.

Briefly centrifuge glass vials before loading — sample can collect on walls and reduce available volume for injection.
9 CoSAXS BioCube measurements ▶

Dilute to 1 mg/mL in PBS. Detector distance 3.5 m. Measurement sequence per sample: Sample → Buffer → Buffer → Drying, then repeat for the next sample.

RiboGreen analysis

Standard curve fitting · EE% · recovery · replicate QC · multi-file batch processing

What the script does

Reads one or more Excel files (each containing a sheet named Data_for_Python), fits a linear standard curve per plate and replicate, calculates mRNA concentrations for all sample wells, and computes encapsulation efficiency and recovery. Outputs plots and an Excel/CSV results table.

Required Excel sheet format — sheet name: Data_for_Python

ColumnTypeDescription
platetextPlate name / condition, e.g. TE or TX
welltextWell identifier, e.g. A1. Row letter is used to group replicates.
kindtextOne of: standard, blank, sample, exclude
std_conc_ng_mlnumberStandard concentration in ng/mL. Leave blank for sample and blank wells.
dilution_xnumberDilution factor applied to the sample before measurement, e.g. 4000
rfunumberMeasured fluorescence value (RFU)
sampletextSample name, e.g. F3 or F3C
fractionnumberFraction number from AF4 or other separation
buffertextBuffer name, e.g. PBS
notetextOptional free-text note (flags, observations)
Download Excel template

How to run

  1. Open Ribogreen_FINAL_average_peak_plots.py in your editor.
  2. Edit the FILES list at the top to point to your Excel files.
  3. Set OUTDIR to the folder where you want plots and tables saved.
  4. Adjust MIN_REPLICATE_R2_FOR_AVG (default 0.99) if needed.
  5. Run: python Ribogreen_FINAL_average_peak_plots.py

Outputs

  • Standard curve plots per plate (with individual replicate lines)
  • Sample concentration plots grouped by fraction
  • Encapsulation efficiency (EE%) curve per sample type
  • ribogreen_clean_output_table.xlsx — full results with flags
  • curve_summary.csv — fitted slope, intercept, R² per plate

Key settings in the script

FILES = [Path("your_file.xlsx"), ...] # list of Excel files SHEET = "Data_for_Python" # sheet name (do not change) OUTDIR = Path("output_folder") # where to save results MIN_REPLICATE_R2_FOR_AVG = 0.99 # min R² for a replicate to be used SATURATION_RFU = 260000 # RFU above this = flagged as saturated CLIP_NEGATIVE_CONC_TO_ZERO = True # set negative concentrations to 0 EE_VALID_MIN = 0 # drop EE% below this (%) EE_VALID_MAX = 100.0 # drop EE% above this (%)

AF4 chromatogram plotter

Reads AF4 instrument Excel exports · plots UV 260 nm and/or Rh traces · auto-detects samples from header

What the script does

Parses the header row of an AF4 Excel export to detect sample names and available channels (UV / Rh), then plots the selected chromatograms with peak annotations and dotted vertical lines at each UV peak maximum.

Required input format — direct AF4 instrument export (.xlsx)

The script reads the Excel file as exported directly by the AF4 instrument software. No manual reformatting needed. The header row must contain column names in the format:

time (min) | 1126 - F3 empty 200uL Vcf175 Vx250g Vf000 (UV) | time (min).1 | 1126 - F3 empty 200uL ... (Rh(Q)) | ...

Duplicate time (min) columns are renamed automatically by pandas (.1, .2, …). The script pairs each UV/Rh column with its preceding time column.

How to run

  1. Open AF4_UV_Rh_plotting_FINAL.py in your editor.
  2. Set the xlsx_path variable inside main() to your file path.
  3. Run: python AF4_UV_Rh_plotting_FINAL.py
  4. The terminal will list detected samples — enter numbers to select, or press Enter for all.
  5. Choose channel: UV, Rh, or both.

Outputs

  • Interactive matplotlib figure (UV and/or Rh overlays)
  • PNG file saved next to the input Excel file (_af4_plot.png)
  • Peak time annotated in legend labels for UV channel

DLS intensity distribution

Log-scale bar chart of particle size distribution from Zetasizer intensity data

What the script does

Takes tab-separated diameter / intensity data from the Zetasizer and plots a log-scale histogram with geometric bin widths. Non-zero bins are highlighted; the main peak is annotated with its diameter.

Required input format — tab-separated text

Copy the size distribution table from your Zetasizer export and paste it as a Python string inside the script. The format is:

Diameter(nm) % Intensity 0.038 0 ... 87.817 1.10632 109.568 79.6935 136.706 19.2001 170.565 0

Two columns, tab-separated, with a one-line header. The full diameter grid (including zero-intensity bins) must be included to preserve the correct log-spacing.

How to run

  1. Open DLS.py in your editor.
  2. Replace the data_form0 string with your Zetasizer data.
  3. Update the title in plot_dls_binned(data_form0, title="...").
  4. To plot multiple formulations, duplicate the data string and the function call.
  5. Run: python DLS.py

Outputs

  • Interactive matplotlib bar chart (log x-axis, diameter in nm)
  • Peak diameter printed to terminal

ChromIXS frame processing

Buffer/sample region detection · spike rejection · buffer subtraction · averaged block export

What the script does

Reads a folder of individual SAXS frames (.dat files), builds an intensity-vs-frame chromatogram trace, and lets you select buffer and sample regions (automatically, manually, or interactively with a slider). It then averages buffer frames (with spike rejection), subtracts the buffer from each sample block, and saves the subtracted curves as new .dat files.

Required input — folder of .dat files

Each file is one SAXS frame in three-column format:

# optional comment lines q I(q) error 1.234e-02 5.678e+00 1.234e-01 ...

Files are read in natural (numerical) order by filename. The q grid must be identical across all frames.

How to run

  1. Open Chromixs.py and set FOLDER to your data directory.
  2. Choose SELECTION_MODE: "auto" (recommended first try), "manual", or "slider".
  3. Set WINDOW_SIZE (frames per averaged output curve, default 50) and STEP (default 50 = non-overlapping blocks).
  4. Run: python Chromixs.py
  5. If using "auto" or "manual", a confirmation plot is shown — press Enter to accept or r to switch to the interactive slider.

Key settings

FOLDER = Path("your/data/folder") WINDOW_SIZE = 50 # frames per output curve STEP = 50 # 50 = non-overlapping; 1 = sliding window SELECTION_MODE = "auto" # "auto" | "manual" | "slider" EXPORT_REMAINDER = True # include last incomplete block? # Auto-detection tuning AUTO_BASELINE_FRACTION = 0.15 # fraction of initial frames for baseline AUTO_PEAK_FRACTION = 0.005 # threshold = baseline + this × (peak − baseline) AUTO_SMOOTH_WINDOW = 30 # smoothing window before thresholding BUFFER_SPIKE_SIGMA = 5.0 # exclude buffer frames > median + N×std

Outputs

  • Subtracted .dat files saved to a new subfolder (_subtracted_50frame_blocks)
  • Each file named with sample and buffer frame ranges
  • Summary chromatogram plot with error bars saved as PNG

Guinier analysis

Automatic Guinier range fitting · Rg ± error · I₀ · qRg validation · batch over folder

What the script does

Reads all .dat files in a folder, searches the low-q region of each for the best Guinier window (maximising R², number of points, and qRg close to 1.0 while keeping qRg,max < 1.3), fits a weighted linear regression to ln(I) vs q², and extracts Rg ± error and I₀. Results are saved to a CSV table.

Required input — folder of .dat files

Same three-column format as ChromIXS (q, I, error). Files are processed in natural numerical order. Only positive-intensity data points are used.

q I(q) error 1.234e-02 5.678e+00 1.234e-01 ...

How to run

  1. Open Guinier.py and set FOLDER to your folder of subtracted frames.
  2. Adjust Guinier criteria if needed (see settings below).
  3. Run: python Guinier.py
  4. Results are printed to the terminal and saved to guinier_results.csv in the same folder.

Key settings

FOLDER = Path("your/subtracted/folder") MIN_POINTS = 6 # minimum points in Guinier window MAX_POINTS = 30 # maximum points in Guinier window QRG_MAX_LIMIT = 1.30 # strict Guinier criterion (qRg,max must be < this) LOWQ_FRACTION = 0.25 # only search lowest 25% of q-range MIN_R2 = 0.995 # minimum R² for a valid window

Outputs

  • guinier_results.csv — Rg, Rg_err, qmin, qmax, qRg_max, I₀, R² per file
  • Terminal summary of all fits (OK / failed with reason)

SAXS offline batch plotter

Plots I(q) curves for a list of buffer-subtracted SAXS files · log-log scale · colour-coded by formulation and buffer

What the script does

Reads a user-defined list of .dat file names from a folder and plots all I(q) curves on a single log-log figure. Colours and markers are assigned automatically based on sample name conventions (F3 vs F3C, PBS vs TRIS).

Required input — folder of .dat files

Buffer-subtracted SAXS files in two-column format (q, I) or three-column (q, I, error). Comment lines starting with # are skipped.

q I(q) 1.234e-02 5.678e+00 ...

File names should follow the convention SampleName_Buffer_scan-XXXXX_eiger_sub.dat for automatic colour assignment.

Naming convention for auto-colouring

Name patternMarkerColour
F3_empty_PBS_...circlelight blue
F3_empty_TRIS_...circledark blue
F3_mRNA_PBS_...circlelight orange
F3_mRNA_TRIS_...circledark orange
F3C_empty_PBS_...squarelight green
F3C_empty_TRIS_...squaredark green
F3C_mRNA_PBS_...squarelight purple
F3C_mRNA_TRIS_...squaredark purple

How to run

  1. Open Offline_batch_samples.py and set FOLDER to your data directory.
  2. Edit FILES_TO_PLOT — list the file stems (without extension) you want to include.
  3. Set EXTENSION = ".dat" (or ".txt" as appropriate).
  4. Set USE_LOGLOG = True for log-log, False for linear.
  5. Run: python Offline_batch_samples.py

Outputs

  • Interactive matplotlib figure with all selected I(q) curves
About

I work on the biophysical characterization of lipid polymer nanoparticles (LPNs) for mRNA delivery, with a focus on formulation, size, structure, and encapsulation efficiency.

This site collects the protocols and analysis scripts developed during my work at the Department of Pharmacy, University of Copenhagen. Methods include microfluidic LPN assembly, AF4 fractionation, SAXS, RiboGreen assays, and custom Python tooling for data processing and visualization.

If you have questions about any of the methods, would like access to the scripts, or are interested in collaborating, please get in touch.
